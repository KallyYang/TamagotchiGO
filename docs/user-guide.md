# Tamago 用户使用文档

## 1. 这是什么

Tamago 是一个在浏览器中运行的拓麻歌子模拟项目。你不需要自己启动前端工具链，只要准备好 Node.js，然后双击启动脚本即可使用。

## 2. 使用前准备

开始前请确认：

- 你的电脑是 macOS 或 Windows
- 已安装 Node.js 18 或更高版本
- 首次启动时可以联网

说明：

- 第一次启动可能会自动执行 `npm install`，速度取决于网络环境
- 启动后会在本地打开一个浏览器页面，同时保留一个启动窗口作为本地服务
- 使用期间不要关闭这个启动窗口；关闭后页面将无法继续访问

## 3. 一键启动

### macOS

1. 打开项目根目录
2. 双击 `start-mac.command`
3. 等待浏览器自动打开

如果浏览器没有自动弹出，请手动打开启动窗口中显示的地址，例如：

```text
http://127.0.0.1:9001
```

### Windows

1. 打开项目根目录
2. 双击 `start-windows.bat`
3. 等待浏览器自动打开

如果 `9001` 端口已被占用，程序会自动改用下一个可用端口，例如 `9002`、`9003`。

## 4. 关闭程序

- 关闭浏览器页面不会停止本地服务
- 要彻底退出，请关闭启动窗口，或在启动窗口中按 `Ctrl+C`

## 5. 页面功能说明

启动成功后，页面顶部和设备界面中可以使用这些功能：

- `无芯片 / Makiko / Shimashimatchi / 拖入芯片文件`
  - 用于切换人物芯片
  - 选择“拖入芯片文件”后，可以把 `.bin` 文件拖到设备区域中载入
- `速度`
  - 可切换为 `1x / 2x / 4x / 8x / 16x`
- `开启声音 / 关闭声音`
  - 控制模拟器声音输出
- `寄存器日志`
  - 打开后可以看到声音、SPI 和部分硬件事件日志，适合排查问题时使用
- `导出存档`
  - 会下载一个存档文件，默认文件名是 `tamago-eeprom-save.json`
- `导入存档`
  - 可把之前导出的存档文件重新导入
- 设备底部 `A / B / C`
  - 可直接点击操作
  - 也可以使用键盘 `A / S / D` 键分别代替 A、B、C

## 6. 存档说明

项目会把进度保存在浏览器本地存储中，所以：

- 同一台电脑、同一个浏览器里再次打开时，通常会保留之前的进度
- 如果你换浏览器、清理浏览器数据，或换一台电脑，本地存档可能不会保留

建议：

- 在重要进度前使用“导出存档”做备份
- 迁移到另一台电脑时，先导出，再在新环境中使用“导入存档”

## 7. 常见问题

### 双击后没有启动

请先检查：

- 是否已安装 Node.js 18+
- 是否误删了项目里的文件
- 首次启动时网络是否可用

如果需要手动确认 Node 版本：

```bash
node -v
```

### 浏览器页面打不开

请确认：

- 启动窗口仍然开着
- 启动日志中显示了实际访问地址
- 你访问的是 `http://127.0.0.1:端口号`，而不是直接双击 `web/index.html`

注意：本项目不能直接通过 `file://` 方式打开 `index.html`，必须通过启动脚本提供的本地服务访问。

### 首次启动很慢

这通常是正常现象，因为程序可能正在自动安装依赖。等启动窗口里的安装和构建过程结束后，浏览器会自动打开。

### 声音没有播放

可以按下面顺序检查：

- 页面上的声音按钮是否处于开启状态
- 浏览器标签页是否被静音
- 是否已经和页面进行过一次点击或按键交互

### 想手动启动

如果你不想双击脚本，也可以在项目根目录手动运行：

```bash
npm install
npm run build:web
npm run start:app
```

## 8. 给使用者的建议

- 把整个项目文件夹保留在本地，不要只单独拷贝某一个 HTML 文件
- 第一次使用时先等启动完成，再进行人物芯片切换和存档操作
- 如果要长期保留进度，记得定期导出存档
